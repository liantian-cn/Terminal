# 颜色约定

颜色是 `Terminal` 解码时必须理解的协议字段，不是随意推测的视觉效果。

## 基础颜色

- `BLACK`: 空白、关闭、未激活
- `WHITE`: 开启、可见、显著布尔状态
- `TRANSPARENT`: 无内容

## 语义颜色组

来自 `DejaVu/02_core/05_color.lua` 的主要分类:

- `SPELL_TYPE`
  - 区分魔法、诅咒、疾病、中毒、激怒、流血
  - 区分友方增益、友方减益、敌方减益、玩家技能、敌方可打断/不可打断施法
- `ROLE`
  - 坦克、治疗、输出、无角色
- `CLASS`
  - 各职业固定色
- `STATUS_BOOLEAN`
  - 这一组没有独立颜色语义
  - 只用于表达布尔值
  - 这一组像素区域为黑色时是 `false`，不是黑色时就是 `true`
  - 不需要区分它具体是哪一种非黑色
- `MARK_FRAME`
  - 只是美观组件
  - 不是锚定区域
  - 解码时务必忽略这一组颜色
- `MARK_POINT`
  - 四角近黑色标记点，用来帮助矩阵定位

## 锚点说明

- 当前锚点信息不以这份颜色文档为准
- 所有锚点都在一份 Excel 文档中
- 这份 Excel 目前还没有提供给 agent
- 以后由用户在对话中输入或补充
- 在拿到那份 Excel 之前，不要把 `MARK_FRAME` 误当成锚点规则

## 解码原则

- 不要把颜色只看成“好看”
- 颜色本身就是协议字段
- 先按“颜色类别”识别，再做状态还原
- `STATUS_BOOLEAN` 只按黑/非黑做布尔判断
- `MARK_FRAME` 务必忽略，不参与锚定、定位或分区判断
- 真正锚点以后以用户提供的 Excel 为准
