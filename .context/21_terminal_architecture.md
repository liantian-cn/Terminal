# Terminal 架构草图

当前适合按下面几层理解，不要求一次性全实现。

## 1. 采集层

- 用 `mss` 抓取屏幕
- 定位 `DejaVu` 矩阵区域
- 输出原始图像帧

## 2. 归一化层

- 把截图转换成 `numpy` 数组
- 做尺寸、坐标、颜色上的基础规整
- 为后续切块提供统一输入

## 3. 解码层

- 按 Matrix 网格切分 `Cell` / `MegaCell` / `BadgeCell` / `CharCell`
- 识别固定标记点和固定标记框
- 还原每个位置的像素类型和原始值

## 4. 识别层

- 图标走哈希索引
- 相似图标结合 OpenCV 做稳健识别
- 颜色映射回协议语义
- 文本块按字体协议还原

## 5. 状态层

- 把矩阵信息还原成结构化对象
- 区分稳定锚点、动态状态和未识别内容

## 6. 上层使用者

- GUI 调试展示
- 策略脚本
- 外部按键执行

## 架构原则

- 先做“稳定还原”，再做“策略执行”
- 先做协议驱动，再做业务推断
- 不把当前 `84 x 28` 尺寸写死成永久常量
